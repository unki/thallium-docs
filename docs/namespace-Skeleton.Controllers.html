Classes summary

AuditController AuditController
ConfigController ConfigController handles loading configuration files.
DatabaseController

DatabaseController handles all the database specific tasks like storing and retriving something from the database.
DefaultController

DefaultController is an abstract class that is used by all the other Thallium Controllers.
HttpRouterController

HttpRouterController parses client requests and takes in user data that is provided by $_GET, $_POST, etc.
InstallerController

InstallerController is mainly used to install the database table schema into the configured database.
JobsController

DefaultController is an abstract class that is used by all the other Thallium Controllers.
MainController

This is the main class of Thallium. MainController initials all the other classes containing Controllers, Models and Views.
MessageBusController

MessageBusController manages receives and sends messages to clients.
PagingController

PagingController acts similar as the PEAR class Pager. On providing data to the controller, it also the split this data into pages so that long lists can be paged.
RequirementsController

RequirementsController is used to check if all the software requirements as well as required directories to be available with the right permissions.
RpcController

RpcController handles remote-procedure-call requests made by clients. This should only be used seldomly and MessageBus is a better joice.
SessionController SessionController handles the PHP session information
TemplatesController

TemplatesController manage the interaction with the Smarty template engine. Usually is accessed by ViewsController or Views themself.
ViewsController

ViewsController takes care of loading views and mapping the clients page requests to a specific view.

ExceptionController

Despite its name, ExceptionController class is not based on Thalliums DefaultController but rather extends PHPs own Exception class.
